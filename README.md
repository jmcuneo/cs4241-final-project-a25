# Details:

For my final project, as a compromise of having to work on my own, I decided to expand upon my Game Review Site that I made for Assignment Three. The main thing I added was, in addition to the normal review page, a game/review "Cluster," where the user could see a breakdown of all of the reviews they've posted by each subscore (gameplay, story, visuals, music, and overall), viewable in the form of a bar graph that can be sorted alphabetically, by date posted, by release year, and by each type of score (ascending and descending).

In other additions, I expanded the user account to both be viewable on its own page and contain more information, specifically their pronouns, bio, and five favourite games, all of which can be edited. I also overhauled the site to make not only other user's account pages but also their review and cluster pages viewable to any user (without of course being editable by anyone other than them), accessible from its own page. Finally, whilst I didn't do anything too crazy to update the UI, I did make sure to update the navigation bar to be more visually appealing, more space economic, and a bit more user-accessible.

As far as technology, much of it already existed in some form that I merely updated, such as adding new user data for Node to keep track of and adding the new requests (account/username, game-cluster/username, users, and the updated reviews/username), to the express server.
However, the main new technology I added was D3, which was of course used for the cluster, both in terms of appearances and its interactivity.

The biggest challenge was unexpectedly the combination of working alone and time management. I tried to account for this as best I could, but it was of course a struggle nonetheless. I am disappointed I wasn't able to find the time to implement the likes feature, but at the same time I'm just relieved I was able to get the assignment done.
